# 自动化主题摘要页面生成项目 - 6人小组工作分配

## 项目概述
构建一个数据管道，自动爬取特定事件或主题的信息，生成包含主要摘要、关键实体、时间线和原始文章链接的静态HTML页面。

---

## 成员1：数据爬取模块 (Data Crawling Module)

### 主要职责
- 设计和实现网页爬虫系统
- 从多个新闻源抓取数据

### 具体任务
1. 使用 BeautifulSoup/Scrapy 构建爬虫
2. 实现多源新闻网站的爬取（至少3-5个新闻源）
3. **实现主题搜索功能** - 支持用户手动输入特定事件或主题
4. 处理网站反爬虫机制（请求头、延时、代理等）
5. 提取文章标题、内容、发布时间、URL等信息
6. 实现增量爬取和定时任务
7. 数据存储到PostgreSQL数据库和文件

### 交付物
- `scraper.py` - 爬虫主程序（支持固定源爬取）
- `topic_search.py` - **主题搜索爬虫**（支持手动输入主题）
- `db_utils.py` - 数据库读取工具
- `init_database.py` - 数据库初始化脚本
- `config.json` - 爬虫配置文件（URL列表、参数等）
- 爬取的原始数据文件（JSON）和PostgreSQL数据库

---

## 成员2：数据清洗与去重模块 (Data Cleaning & Deduplication)

### 主要职责
- 清洗和标准化爬取的数据
- 处理重复内容和合并信息

### 具体任务
1. 数据清洗：去除HTML标签、特殊字符、广告内容
2. 文本标准化：统一日期格式、编码转换
3. 重复检测算法实现（基于内容相似度）
4. 合并来自不同源的相同新闻
5. 数据质量验证和异常处理
6. 输出清洗后的结构化数据

### 交付物
- `data_cleaner.py` - 数据清洗模块
- `deduplicator.py` - 去重算法实现
- 清洗后的数据文件

---

## 成员3：实体识别与提取模块 (Entity Recognition & Extraction)

### 主要职责
- 使用NLP技术识别关键实体
- 提取人物、组织、地点等信息

### 具体任务
1. 集成NER（命名实体识别）模型
   - 可使用 spaCy、NLTK 或 Transformers
2. 识别和提取：
   - 人物 (People)
   - 组织 (Organizations)
   - 地点 (Locations)
   - 日期 (Dates)
3. 实体频率统计和排序
4. 实体关系提取（可选高级功能）
5. 生成结构化实体列表

### 交付物
- `entity_extractor.py` - 实体提取模块
- `entities.json` - 提取的实体数据
- 实体统计报告

---

## 成员4：文本摘要生成模块 (Text Summarization)

### 主要职责
- 使用LLM API生成主要摘要
- 创建事件时间线

### 具体任务
1. 集成LLM API（OpenAI、Claude、或开源模型）
2. 设计提示词(Prompt)生成高质量摘要
3. 实现多文档摘要合成
4. 从时间戳信息生成时间线
5. 处理API调用限制和错误重试
6. 优化摘要质量和一致性

### 交付物
- `summarizer.py` - 摘要生成模块
- `timeline_generator.py` - 时间线生成
- `summary_output.json` - 摘要和时间线数据

---

## 成员5：HTML页面生成模块 (HTML Page Generation)

### 主要职责
- 设计和实现静态HTML页面
- 创建美观的展示界面

### 具体任务
1. 设计HTML模板结构
2. 使用Jinja2或类似模板引擎
3. 实现页面布局：
   - 主摘要区域
   - 关键实体列表
   - 时间线展示
   - 原始文章链接
4. CSS样式设计（响应式布局）
5. 添加JavaScript交互功能（可选）
6. 生成最终静态HTML文件

### 交付物
- `html_generator.py` - HTML生成器
- `templates/` - 模板文件夹
- `static/` - CSS/JS资源文件
- `output.html` - 最终生成的页面
